üß™ Test Checklist

1. Settings Persistence

  • Fill out all form fields (budget, people, meals)
  • Click "Save Settings" button
  • Refresh the page (F5)
  • Click "Load Settings" button
  • Verify all fields are populated correctly
  • Check toast message appears

2. Breakfast/Lunch/Dinner Sections

  • Set breakfast: 7 days, 2 people, 3 recipes
  • Set lunch: 5 days, 1 person, 2 recipes
  • Set dinner: 7 days, 4 people, 5 recipes
  • Select cuisine preferences (e.g., Italian, Asian)
  • Choose cooking skill level
  • Select health focus
  • Generate meal plan
  • Verify AI creates separate breakfast/lunch/dinner recipes

3. Shopping List Manager

  • Generate a meal plan
  • Save shopping list with custom name
  • Click "View Saved Lists" button
  • Verify list appears in modal
  • Check/uncheck items (verify progress bar updates)
  • Download list as text file
  • Print list (check print preview)
  • Delete list with confirmation dialog

4. Product Descriptions

  • Generate meal plan
  • Open shopping list
  • Verify NO vague items (no "3 slices" or "0.63 oz")
  • Check all items have specific sizes (e.g., "16 oz bag")
  • Verify brand-style descriptions
  • Check spices have jar sizes (e.g., "2.5 oz jar")

5. AI Creativity (No Repetition)

  • Generate meal plan #1 (note recipes)
  • Generate meal plan #2 (verify different recipes)
  • Generate meal plan #3 (verify different recipes)
  • Confirm no repetitive meals across generations
  • Check recipes match cuisine preferences

6. Mobile Responsiveness

  • Open DevTools (F12) and toggle device toolbar
  • Test on mobile viewport (375px width)
  • Verify settings buttons stack vertically
  • Check form sections are single column
  • Test shopping list modal on mobile
  • Verify all buttons are touch-friendly
